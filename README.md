# 💳 Credit Card Fraud Detection

## 📌 Problem Statement
Detect fraudulent transactions from highly imbalanced financial data.

---

## 📊 Dataset
Transaction dataset containing features like transaction amount, location, and population data.

---

## ⚙️ Steps Performed
- Data Cleaning and Feature Selection  
- Handling Large Dataset with Memory Optimization  
- Handling Extreme Class Imbalance  
- Model Training and Evaluation  

---

## 🤖 Models Used
- Logistic Regression  
- Random Forest  

---

## 📈 Results

| Model | Recall (Fraud) | Accuracy |
|------|---------------|----------|
| Logistic Regression | **0.76 ✅** | Lower |
| Random Forest | 0.03 ❌ | Higher |

---

## 🧠 Key Insights
- Accuracy is misleading in fraud detection problems  
- Recall is the most important metric for identifying fraud  
- Class imbalance significantly impacts model performance  

---

## 🚀 Future Improvements
- Apply SMOTE for better balancing  
- Use XGBoost for improved performance  
- Deploy as a fraud detection API
